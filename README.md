# AoC2022
Solutions to 2022's Advent of Code
by Dan Higdon

This year, I'm forgoing my customary Common Lisp or ANSI-C explorations in favor of a more limited language - GWBASIC.
The memory restrictions of this platform could prove challenging, but I plan to use intermediate files to help with that.
If we wind up needing BigNums, I may have to give up on GWBASIC. We shall see.
